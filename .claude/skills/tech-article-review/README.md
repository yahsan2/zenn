# 技術記事推敲スキル

技術記事の品質を向上させるための体系的な推敲を支援するClaude skillとコマンドです。

## 構成

### 1. Skill: `tech-article-review`

**場所**: `.claude/skills/tech-article-review/SKILL.md`

技術記事を6つの観点で体系的にチェックし、4つの推敲技法を適用します。

#### 6つのチェック観点

1. **記事構造** - 導入・本文・まとめのバランスと論理的な流れ
2. **読みやすさ** - 文章の長さ、段落構成、視覚的な読みやすさ
3. **技術的正確性** - 用語、コード、バージョン情報の正確性
4. **表現の明確さ** - 曖昧な表現の排除、主語の明確化
5. **コードサンプルの品質** - 可読性、完全性、説明の充実度
6. **ターゲット読者への適切さ** - 難易度と前提知識の適切性

#### 4つの推敲技法

石原尚氏（大阪大学）の推敲手法を技術記事に適用:

- **切り分ける** - 長い文章や段落を適切に分割
- **並び替える** - 論理的な順序に再構成
- **繋ぎ方を変える** - セクション間の接続を改善
- **削り落とす** - 冗長な表現を簡潔に

参考: [果てしない文章推敲を『３ステップ』で効率的に終わらせる方法](https://note.com/hisashi_is/n/n2efd3f4bb09a)

### 2. Slash Command: `/review-article`

**場所**: `.claude/commands/review-article.md`

Taskツールを使用してsubagentを起動し、tech-article-review skillを実行するコマンドです。

## 使い方

### 方法1: Slash Commandを使用（推奨）

```bash
/review-article articles/my-article.md
```

または引数なしで実行すると、ファイルパスを尋ねられます:

```bash
/review-article
```

### 方法2: 直接依頼

```
articles/my-article.md を推敲して
```

または:

```
記事をレビューして
```

## 出力フォーマット

推敲結果は以下の構造で提示されます:

- **総合評価**: 記事の現状と全体的な印象
- **主な改善ポイント**: 6つの観点別の具体的な指摘と改善案
- **推敲の4技法による具体的修正案**: 行番号付きの修正提案
- **優先的に対応すべき項目TOP3**: 重要度の高い改善点
- **良い点**: 記事の優れているポイント

## カスタマイズ

### Skillのカスタマイズ

`.claude/skills/tech-article-review/SKILL.md`を編集して:

- チーム固有のライティングガイドラインを追加
- 特定のプラットフォーム（Zenn、Qiitaなど）向けのチェック項目を追加
- 出力フォーマットの調整
- チェック観点の重み付け変更

### Commandのカスタマイズ

`.claude/commands/review-article.md`を編集して:

- デフォルトの記事ディレクトリを設定
- 追加の前処理・後処理を追加
- レポート形式のカスタマイズ

## 技術スタック
